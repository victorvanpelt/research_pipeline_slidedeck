# AGENTS.md — context for AI coding agents

This repository holds one Quarto slide deck: `presentation.qmd` renders to `presentation.pdf` on the WHU beamer template. There is no analysis pipeline here. The pipeline the slides talk about lives in its own repository: https://github.com/victorvanpelt/research_pipeline_example.

## Rules

- Edit `presentation.qmd`, and `materials/references.bib` when citations change. Everything else is supporting material.
- Never edit `presentation.pdf` or `presentation.tex` by hand: both are build artifacts, regenerated by `make`.
- Treat `images/` as fixed inputs. Most files are screenshots and renders of the companion repository; do not redraw, recompress, or rename them. Adding a new image for a new slide is fine.
- Do not change `materials/talk_template.sty`, the fonts, or anything else in `materials/` unless that change is the explicit task.
- The template drops some math glyphs; write statistics and symbols in plain ASCII in slide text.
- The deck uses `date: today`, so every render restamps the title slide. A changed date alone is not a content change.
- Before a change that touches more than one file, say which files you will edit and what will change in each. Then edit.
- Never run `git commit` or `git push`. Leave changes in the working tree for the maintainer to review and commit.
- Remove any scratch or helper files you create. The folder structure is fixed; do not add folders or top-level files.

## Build

- Render the slides: `make` (runs `quarto render presentation.qmd`)
- Delete the rendered PDF and the intermediate `.tex`: `make clean`

## Verification

A change is done only when `make` completes without errors and `presentation.pdf` was regenerated.

Verify before you report:

1. After the render, check that `presentation.pdf` is new and open the pages you changed.
2. Run `git status` and confirm that only the files you meant to change are modified.
3. Report what changed, whether the render succeeded, and anything you could not verify.

Do not report a change as done without having rendered. "It should work" is not verification.
